# Soul Injection UI Refactor Spec

## Why
为了增强用户的情感参与度和沉浸感，弥补最终生图可能存在的物理瑕疵，需要将“阶段二”枯燥的特征标签展示重构为类似“RPG 游戏角色建立”或“赛博宠物领养登记”的互动界面，让用户深度参与文本共创，同时在“阶段四”最终展示时进行对应的情感化渲染。

## What Changes
- 重构阶段二（特征确认）表单界面，引入“专属命名”、“灵魂性格”和“AI视觉底稿”三大模块。
- 增强阶段三（图像生成）组装逻辑，以特定顺序拼接用户填写的性格、AI提取的物理特征与补充特征。
- 重构阶段四（结果展示）页面，移除过时的“升级/排队/Pro”相关假门测试按钮，改为显示宠物名字的专属动态标题，并添加“再试一次”与“保存它的赛博档案”按钮。
- **BREAKING**: 前端传给后端的生图请求负载需带上新组装的特征数据或后端需支持接收更详细的字段。

## Impact
- Affected specs: 宠物克隆全流程的用户交互体验。
- Affected code:
  - 前端界面层（`app/page.tsx` 或相关组件，表单与结果页UI）
  - 后端接口层（`app/api/clone/route.ts`，组装终极 Prompt 逻辑）

## ADDED Requirements
### Requirement: 建立宠物赛博档案
The system SHALL provide 包含以下模块的表单以建立赛博档案：
1. 专属命名（必填）：提供一个强引导的输入框。
2. 灵魂性格：支持用户输入自定义的性格/氛围词。
3. AI 视觉底稿：以胶囊 Tag 形式展示 Qwen3-VL 识别出的物理特征，支持增删编辑。

#### Scenario: 组装终极 Prompt
- **WHEN** 用户填写完毕并点击“开始生成平行宇宙的它”
- **THEN** 提取出的最终特征结构为："{灵魂性格} 的氛围与神态, {AI物理特征Tags}, 补充特征: {补充特征}"，并以此组装绘图 Prompt 提交至后端。

## MODIFIED Requirements
### Requirement: 结果页情感化展示
- 结果页图卡标题需结合宠物命名动态渲染（如：“盲盒宇宙里的 {petName}”）。
- 底部操作区调整为“再试一次”或“保存它的赛博档案”等情感化操作按钮。

## REMOVED Requirements
### Requirement: 假门测试逻辑
**Reason**: 与新的沉浸式赛博领养体验和结果页设计冲突，且之前仅用于验证转化。
**Migration**: 彻底移除所有相关的“升级、排队、Pro版”按钮及其弹窗逻辑。
